## 🧭 Sobre mí

Soy un desarrollador orientado principalmente al **frontend con Angular**, con un enfoque fuerte en construir aplicaciones web que puedan crecer de forma ordenada, clara y mantenible.

Me interesa trabajar en desarrollos donde la calidad visual, la arquitectura y la lógica del negocio tengan el mismo peso.  
Por eso, una parte importante de mi trabajo está en diseñar estructuras que no solo funcionen hoy, sino que sigan teniendo sentido cuando el sistema evolucione.

Actualmente, uno de mis principales intereses es aplicar principios como **Screaming Architecture** en proyectos Angular, buscando que la estructura del código refleje el dominio y las funcionalidades del producto antes que la herramienta en sí.

---

## 🏛️ Cómo me gusta construir software

### 🔹 Arquitectura con intención
Me importa que los proyectos tengan una base sólida, con una estructura fácil de entender, de escalar y de mantener con el tiempo.

### 🔹 Angular como base de soluciones reales
No uso Angular solo para construir interfaces, sino para desarrollar aplicaciones robustas, modulares y alineadas con necesidades reales de producto.

### 🔹 Código que comunica
Valoro que la organización del proyecto tenga sentido desde el negocio: features claras, responsabilidades separadas y una estructura que hable por sí sola.

### 🔹 Visión más allá del frontend
Aunque mi especialidad está en Angular, también he trabajado con backend, bases de datos, cloud y contenedores, lo que me permite entender mejor la solución completa.
